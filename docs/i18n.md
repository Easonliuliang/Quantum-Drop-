## i18n 指南

Quantum Drop 的界面文案由 `src/lib/i18n.tsx` 中的 `DEFAULT_MESSAGES` 提供，结构为 `locale -> { key: value }`。默认内置 `zh-CN` / `en` 两个语言包，并在 `LocaleSwitch` 组件里提供选择器。若要扩展多语言或补充文案，可按以下步骤处理：

1. **补充文案键 (key)**  
   - 所有面板/按钮请使用 `t("key", "fallback")` 访问，`fallback` 会在运行期兜底。  
   - 将对应的键值添加到 `DEFAULT_MESSAGES` 的各语言对象中。

2. **新增语言**  
   - 在 `DEFAULT_MESSAGES` 中添加新的语言对象，例如 `fr`。  
   - 在 `SUPPORTED_LOCALES` 数组里追加条目（`value` 对应语言代码，`labelKey` 指向翻译后的名称）。  
   - `LocaleSwitch` 会自动读取并展示该语言。

3. **文案组织建议**  
   - 以语义组织键名，例如 `panel.audit`、`settings.chunk.min` 等。  
   - 文案中若包含变量，使用 `{token}` 占位（例如 `license.quotaUsage`），`t` 函数会自动替换。

4. **调试与校验**  
   - 切换语言后观察 UI 是否实时更新。  
   - `npm run test -- App.test.tsx` 可覆盖语言切换带来的渲染变化（若有相关快照）。  
   - CI 中的 `npm run check` 会同时跑 ESLint/Vitest，确保新增文案不会破坏现有组件。

> 提示：如果需要将文案拆分为独立 JSON/TS 模块，可在 `src/lib/i18n.tsx` 中加载外部资源，或引入如 `i18next`/`react-intl` 等库替换当前的极简实现。
